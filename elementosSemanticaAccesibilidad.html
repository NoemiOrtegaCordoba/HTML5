Elementos de Semántica y Accesibilidad en HTML5

Categoría Elemento/Característica Descripción
Estructura de Documento <header> Define la cabecera de un documento o sección. Contiene información como logotipos y encabezados.
<nav> Define una sección de navegación. Útil para menús de navegación principal o secundaria.
<main> Define el contenido principal de un documento, facilitando la navegación para lectores de pantalla.
<article> Define un contenido independiente y autocontenido, como una publicación de blog o un artículo de noticias.
<section> Define una sección temática dentro de un documento, lo que mejora la estructura y comprensión del contenido.
<aside> Define contenido relacionado pero no esencial, como anuncios o información adicional.
<footer> Define el pie de página de un documento o sección, que suele contener información de contacto y enlaces relacionados.
Elementos de Texto y Títulos <h1>, <h2>, ... Definen encabezados de diferentes niveles, lo que ayuda a estructurar el contenido y mejora la accesibilidad.
<p> Define un párrafo, permitiendo una presentación clara y legible del texto.
<blockquote> Define una cita o bloque de texto destacado, mejorando la presentación de citas y contenido citado.
Listas y Elementos de Definición <ul>, <ol> Definen listas no ordenadas y ordenadas, ideales para presentar información en formato de lista.
<li> Define un elemento de lista, contribuyendo a una presentación ordenada de elementos.
<dl> Define una lista de definición, útil para glosarios y términos con sus descripciones correspondientes.
<dt> Define un término en una lista de definición, ayudando a estructurar glosarios y definiciones.
Elementos de Enlace <a> Define un enlace, permitiendo la navegación a otras páginas o recursos relacionados.
<nav> Define una sección de navegación, que facilita la exploración del sitio web.
<menu> Define un menú de opciones, lo que mejora la usabilidad y navegación en sitios web interactivos.
Elementos de Formularios <form> Define un formulario para la entrada de datos, esencial para la interacción del usuario.
<input> Define un campo de entrada de datos en un formulario, incluyendo opciones como texto, radio, y más.
<label> Etiqueta de texto para un elemento de formulario, mejorando la accesibilidad y experiencia del usuario.
<button> Elemento botón interactivo para formularios, proporcionando una manera clara de enviar información.
Elementos de Contenido Incrustado <figure> Define contenido incrustado, como imágenes, videos o gráficos, junto con una descripción, mejorando la accesibilidad.
<figcaption> Define una descripción para el contenido dentro de un elemento <figure>, facilitando la comprensión para todos los usuarios.
Elementos de Accesibilidad <mark> Resalta texto para llamar la atención, lo que es especialmente útil para lectores de pantalla.
<strong> Indica un texto fuerte o importante, permitiendo una mayor comprensión del contenido.
<em> Enfatiza el texto, lo que puede ser crucial para transmitir énfasis en la lectura en voz alta.
<abbr> Define una abreviatura, proporcionando una explicación adicional para términos abreviados.
<time> Indica una fecha o hora, permitiendo una mejor interpretación de fechas y horarios por parte de los navegadores y lectores de pantalla.
Elementos Multimedia <audio> Permite la inserción de contenido de audio en la página, lo que es esencial para la accesibilidad de personas con discapacidades auditivas.
<video> Permite la inserción de contenido de video en la página, facilitando la inclusión de contenido multimedia.
<canvas> Define un área gráfica en la que se puede dibujar a través de scripts, lo que es útil para aplicaciones interactivas.
Elementos de Tablas <table> Define una tabla, lo que es esencial para mostrar datos tabulares de manera estructurada.
<tr> Define una fila en una tabla, ayudando a organizar y presentar datos de manera ordenada.
<th> Define una celda de encabezado en una tabla, mejorando la accesibilidad y comprensión de datos.
